# ConnectorTool 测试说明

这份文档是给第一次接触测试项目的人看的。你可以把 `ConnectorTool.Tests` 理解成“自动帮你点一遍关键逻辑的检查员”，它不会打开真实界面，也不会真的在屏幕上乱点。

## 测试项目是什么

`ConnectorTool` 是主程序，也就是连点器本体。

`ConnectorTool.Tests` 是测试项目，专门验证主程序里的服务代码有没有按预期工作，比如配置能不能保存、日志是不是只在写入时创建目录、随机扰动范围是不是正确、连点启动和停止状态是否正确。

测试项目引用了主项目：

```xml
<ProjectReference Include="..\ConnectorTool\ConnectorTool.csproj" />
```

所以测试代码可以直接调用主项目里的 `ClickService`、`SettingsService`、`LogService` 等类。

## MSTest 常见写法

`[TestClass]` 表示这是一个测试类。测试运行器会扫描这些类。

`[TestMethod]` 表示这是一个测试方法。每个测试方法通常只验证一个明确行为。

`Assert.AreEqual(expected, actual)` 表示期望值和实际值必须相等。

`Assert.IsTrue(condition)` 表示条件必须为 true。

`Assert.IsFalse(condition)` 表示条件必须为 false。

`Assert.ThrowsException<T>(action)` 表示这段代码必须抛出指定类型的异常。

## 一个简单例子

下面这个测试的意思是：关闭随机扰动时，传入 1000 毫秒，就应该返回 1000 毫秒。

```csharp
[TestMethod]
public void GetDelay_WhenPerturbationDisabled_ShouldReturnBaseInterval()
{
    var service = new ClickTimingService(new Random(1234));

    Assert.AreEqual(1000, service.GetDelay(1000, false));
}
```

测试方法名一般写得比较长，是故意的。推荐格式是：

```text
被测方法_When条件_Should结果
```

这样以后测试失败时，不用点进去看代码，也能大概知道坏在哪里。

## 怎么运行全部测试

在项目根目录执行：

```powershell
dotnet test D:\project\cs_project\ConnectorTool\ConnectorTool.Tests\ConnectorTool.Tests.csproj
```

如果看到类似下面的结果，就表示测试通过：

```text
Passed!  - Failed: 0, Passed: 12
```

## 怎么只运行一个测试

可以用 `--filter` 过滤测试名：

```powershell
dotnet test D:\project\cs_project\ConnectorTool\ConnectorTool.Tests\ConnectorTool.Tests.csproj --filter GetDelay_WhenPerturbationDisabled_ShouldReturnBaseInterval
```

如果你只想跑某个测试类，也可以过滤类名：

```powershell
dotnet test D:\project\cs_project\ConnectorTool\ConnectorTool.Tests\ConnectorTool.Tests.csproj --filter ClickTimingServiceTests
```

## 测试失败怎么看

测试失败时重点看三块信息：

1. 失败的测试名，例如 `SaveAndLoad_ShouldRoundTripSettings`。
2. 失败原因，例如 `Assert.AreEqual failed`。
3. 堆栈里的文件和行号，例如 `SettingsServiceTests.cs:55`。

一般先点到失败行，看“期望值”和“实际值”差在哪里，再判断是代码逻辑错了，还是测试里的预期需要更新。

## 为什么测试里不用真实鼠标点击

真实鼠标点击属于系统行为，不适合单元测试直接执行。否则测试一跑，鼠标就可能乱点屏幕，风险很高。

当前项目的做法是：`ClickService` 支持把“点击动作”作为委托传进去。正式运行时传真实点击，测试时传假的点击函数。这样既能验证启动、停止、异常处理，又不会影响你的电脑操作。

## 怎么新增一个测试

新增测试通常按这几步来：

1. 在 `ConnectorTool.Tests\Services` 里找到对应服务的测试文件。
2. 新增一个带 `[TestMethod]` 的 public 方法。
3. 准备输入数据。
4. 调用被测方法。
5. 用 `Assert` 判断结果。

例如要验证“间隔最小不能低于 1 毫秒”，可以写成：

```csharp
[TestMethod]
public void GetDelay_WhenBaseIntervalIsZero_ShouldReturnAtLeastOne()
{
    var service = new ClickTimingService(new Random(1234));

    var delay = service.GetDelay(0, false);

    Assert.AreEqual(1, delay);
}
```

写测试时不用追求一次覆盖所有情况。一个测试只验证一个点，越清楚越好。
