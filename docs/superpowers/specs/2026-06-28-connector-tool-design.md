# ConnectorTool WinForms 设计文档

## 一、概述

`ConnectorTool` 是一个基于 `.NET Framework 4.8` 的 WinForms 连点器。
核心使用场景是帮助用户在游戏剧情、重复确认弹窗等场景中，减少手动重复点击。

本项目会参考现有 `MouseClickTool-master` 中的部分功能思路，但不会照搬其实现，而是以 WinForms 重新开发一套结构更清晰、行为更可控、UI 更美观的版本，并采用蓝白渐变视觉风格。

## 二、目标

- 开发一款稳定、易用、界面美观的 WinForms 连点器
- 支持固定坐标点击，适配游戏剧情推进场景
- 支持全局热键，在游戏窗口位于前台时仍可开始或停止连点
- 将界面、热键、点击逻辑、配置、日志拆分为独立职责模块，保证代码规范和可维护性
- 支持用户配置持久化，重启后自动恢复

## 三、V1 不做的内容

- 自定义脚本执行
- 启动外部程序
- 滚轮模式
- 长按模式
- 固定点击次数
- 多套配置档管理
- 系统托盘常驻增强功能

## 四、用户场景

### 1. 主要场景

1. 用户启动程序
2. 用户点击 `选择位置`
3. 程序暂时隐藏主窗口，并显示全屏拾取层
4. 用户在游戏目标位置单击一次
5. 程序记录该屏幕坐标，并在主界面显示
6. 用户将程序最小化
7. 用户按下开始热键，程序开始在该固定坐标持续点击
8. 用户按下停止热键，程序停止点击

### 2. 次要场景

1. 用户开启日志记录
2. 用户选择自定义日志目录
3. 程序只记录高价值事件，例如启动、停止、坐标更新、热键注册失败、异常信息

## 五、功能需求

### 1. 点击功能

- 程序仅支持点击用户选定的固定屏幕坐标
- V1 只支持左键点击
- 程序启动连点后应持续无限点击，直到用户主动停止
- 未选择目标坐标时，程序不得启动连点
- 点击间隔由用户以毫秒为单位配置

### 2. 坐标拾取

- 程序需要提供 `选择位置` 操作
- 点击后进入全屏拾取模式
- 用户在屏幕任意位置单击一次后，即完成坐标记录
- 记录结果需要显示在独立的坐标展示区域，例如 `X: 1234, Y: 567`
- 用户按 `Esc` 取消拾取时，应保留之前已保存的坐标

### 3. 开始与停止控制

- 程序需要提供可见的 `开始` 和 `停止` 按钮
- 程序同时提供全局热键来开始和停止
- 按钮和热键必须调用同一套内部逻辑，不能分成两套行为
- 默认热键为 `F1` 开始、`F2` 停止
- 热键必须支持用户自定义
- 如果热键因冲突无法注册，程序需要明确提示，并保留之前可用的热键配置

### 4. 最小化后运行

- 程序最小化后，仍应继续接收已注册的全局热键
- 用户应可以在最小化状态下，仅通过热键控制开始和停止

### 5. 随机扰动

- 程序需要提供 `随机扰动` 开关
- V1 的随机扰动只作用于点击间隔
- V1 的随机扰动不改变点击坐标
- 开启后，每次点击间隔会在用户设置值附近做轻微波动
- 默认扰动范围为 `+-10%`
- 随机扰动默认开启

### 6. 日志功能

- 程序需要提供 `启用日志` 开关
- 程序需要支持用户选择自定义日志目录
- 程序不能记录每一次点击
- 程序只记录以下事件：
  - 程序启动
  - 程序关闭
  - 连点开始
  - 连点停止
  - 坐标更新
  - 热键注册失败
  - 非预期异常

### 7. 配置持久化

- 程序需要持久化以下内容：
  - 点击间隔
  - 开始热键
  - 停止热键
  - 目标坐标
  - 随机扰动开关状态
  - 日志开关状态
  - 日志目录
- 程序下次启动时应自动恢复这些设置

## 六、界面设计

### 1. 布局结构

界面整体采用紧凑型控制面板设计，而不是默认 WinForms 风格的空白窗体。

建议分区如下：

1. 标题与状态区
2. 点击参数配置区
3. 坐标选择区
4. 热键配置区
5. 日志与选项区
6. 底部操作区

### 2. 视觉风格

- 整体背景使用蓝白渐变
- 内容区使用白色卡片样式，带轻微边框和层次感
- 主按钮采用蓝色高亮
- 状态区域区分 `待机`、`就绪`、`运行中`
- 控件间距和对齐方式优于参考项目，整体更整洁

### 3. 关键控件

- 点击间隔输入控件：`NumericUpDown` 或限制型输入框
- 坐标展示框：只读文本框
- `选择位置` 按钮
- 开始热键下拉框
- 停止热键下拉框
- `随机扰动` 复选框
- `启用日志` 复选框
- 日志路径展示框
- `浏览` 按钮
- `开始`、`停止`、`最小化` 按钮

## 七、架构设计

实现上不采用所有逻辑都堆在 `Form1` 中的方式，而是按职责拆分。

### 1. 主要组件

- `MainForm`
  - 负责界面交互、状态展示、控件事件处理
  - 接收热键消息并分发操作

- `ClickService`
  - 负责后台点击循环
  - 负责点击开始、停止、时间间隔和随机扰动

- `HotkeyService`
  - 负责全局热键注册与注销
  - 负责热键切换时的安全更新和冲突处理

- `PositionPickerForm`
  - 负责全屏拾取坐标
  - 在用户单击时返回屏幕坐标

- `SettingsService`
  - 负责配置文件读取和保存

- `LogService`
  - 负责写入结构化日志，仅记录高价值事件

- `NativeMethods`
  - 统一封装 `RegisterHotKey`、`UnregisterHotKey`、`SetCursorPos`、`SendInput` 等 Win32 调用

- `AppSettings`
  - 负责承载程序配置数据

## 八、数据模型

### 1. AppSettings

V1 计划包含以下字段：

- `int ClickIntervalMs`
- `Keys StartHotkey`
- `Keys StopHotkey`
- `int TargetX`
- `int TargetY`
- `bool HasTargetPosition`
- `bool RandomPerturbationEnabled`
- `bool LoggingEnabled`
- `string LogDirectory`

### 2. 运行时状态

运行状态不直接写入配置，而是单独管理：

- `Idle`
- `Ready`
- `Running`
- `PickingPosition`

其中 `Ready` 表示必须的配置已经完整，用户可以立即开始连点。

## 九、技术实现设计

### 1. 全局热键

- 使用 `RegisterHotKey` 和 `UnregisterHotKey`
- 开始和停止分别使用不同的热键 ID
- 在主窗体的窗口消息处理中响应热键消息
- 如果热键注册失败，需要恢复到之前可用的热键配置，并提示用户

明确限制：
Windows 全局热键不能强行覆盖已经被其他程序注册的相同组合，因此程序只能提示冲突，不能保证一定抢占成功。

### 2. 点击执行

- 使用后台任务执行点击循环
- 每轮逻辑如下：
  - 将鼠标移动到目标坐标
  - 通过 Win32 输入接口发送左键点击
  - 按照当前间隔等待下一次点击
- 使用 `CancellationTokenSource` 进行停止控制

### 3. 随机扰动算法

假设用户设置的基础间隔为 `baseMs`：

- 若关闭随机扰动，则直接使用 `baseMs`
- 若开启随机扰动，则每次从 `[baseMs * 0.9, baseMs * 1.1]` 区间内取值
- 结果最小不低于 `1ms`

该设计只扰动时间，不扰动坐标。

### 4. 坐标拾取层

V1 采用全屏拾取层，而不是全局鼠标钩子，因为实现更简单，行为更稳定，也更容易控制取消逻辑。

行为定义：

- 无边框、置顶、全屏窗体
- 使用半透明深色遮罩
- 中央显示提示文案，例如 `请在目标位置单击一次，按 Esc 取消`
- 捕获用户下一次左键单击
- 返回绝对屏幕坐标
- 选取后立即关闭

### 5. 配置存储

配置文件路径：

- `%AppData%\ConnectorTool\settings.json`

选择该路径的原因：

- 不污染注册表
- 属于当前用户目录，隔离性更好
- 便于查看和备份

### 6. 日志设计

日志文件行为：

- 写入用户指定目录
- 按天生成日志文件，例如 `ConnectorTool-2026-06-28.log`
- 使用 UTF-8 追加写入
- 每行包含时间戳和事件类别

事件类型示例：

- `Info`
- `Warning`
- `Error`

## 十、异常处理

- 点击间隔非法时，应在 UI 层直接限制，而不是启动后再报错
- 未选择坐标时点击开始，应提示用户先完成坐标选择
- 热键注册失败时，应明确提示是哪个热键失败
- 日志目录创建失败时，应关闭当前会话的日志功能并提示用户
- 点击循环中若出现非预期异常，应安全停止连点并提示错误

## 十一、测试方案

### 1. 手工测试

- 启动程序并检查默认配置
- 选择一个目标坐标并确认界面显示正确
- 通过按钮开始连点
- 通过按钮停止连点
- 程序最小化后通过热键开始连点
- 程序最小化后通过热键停止连点
- 重启程序并确认配置已恢复
- 更换日志目录并确认日志写入位置正确
- 开启和关闭随机扰动，并确认点击节奏仍然正常
- 未选择坐标时尝试开始，并确认验证提示正常

### 2. 代码级验证

由于当前项目是 `.NET Framework` WinForms 项目，且尚未存在自动化测试体系，V1 主要依赖手工验证，同时对适合独立验证的逻辑尽量抽取：

- 随机扰动间隔计算
- 配置序列化与反序列化
- 日志路径生成

## 十二、实现说明

- 尽量使用职责明确的小类，而不是把所有逻辑都塞进 `Form1`
- 所有 Win32 互操作集中在一个文件中维护
- 点击循环使用异步后台执行，避免阻塞 UI 线程
- V1 保持功能聚焦，以稳定和易用为优先

## 十三、已确认决策

- V1 点击模式：固定坐标
- V1 点击次数：无限点击，直到手动停止
- 开始与停止逻辑：按钮与热键共用同一套逻辑
- 默认开始热键：`F1`
- 默认停止热键：`F2`
- 随机扰动：默认开启，只作用于点击间隔，范围 `+-10%`
- 日志：可选开启，自定义目录，不记录每次点击

## 十四、实现顺序

1. 实现配置模型和基础服务
2. 实现坐标拾取层
3. 实现主界面 UI
4. 接入热键和点击循环
5. 接入日志和配置持久化
6. 完成手工验证
